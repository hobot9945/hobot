# 🧠 ИНСТРУКЦИЯ ДЛЯ AI: ЗАХВАТ RUST-ПРОЕКТА ИЗ ФАЙЛОВОЙ СИСТЕМЫ

Ты — AI-ассистент, который должен **полностью прочитать и запомнить структуру Rust-проекта** для последующего анализа, доработки или отладки. Проект находится на Windows.

## 🔧 КАК РАБОТАТЬ

### 1. ПОЛУЧИТЬ СТРУКТУРУ ПРОЕКТА
Сначала выполни команду, чтобы увидеть дерево файлов:
```json
{
    "name": "shell_cmd",
    "params": ["tree /F /A \"C:\\путь\\к\\проекту\""]
}
```
Или если `tree` недоступен:
```json
{
    "name": "powershell_cmd",
    "params": ["Get-ChildItem -Recurse -File \"C:\\путь\\к\\проекту\" | Select-Object FullName"]
}
```

### 2. ПРОЧИТАТЬ КЛЮЧЕВЫЕ ФАЙЛЫ ПО ПОРЯДКУ

#### a) `Cargo.toml` (зависимости и настройки)
```json
{
    "name": "shell_cmd",
    "params": ["type \"C:\\путь\\к\\проекту\\Cargo.toml\""]
}
```

#### b) `src/main.rs` (точка входа)
```json
{
    "name": "shell_cmd",
    "params": ["type \"C:\\путь\\к\\проекту\\src\\main.rs\""]
}
```

#### c) Модули из `src/`
Проходи по структуре, начиная с корневых модулей:
- `src/lib.rs` (если есть)
- `src/agent.rs`
- `src/glob.rs`
- и т.д.

#### d) Подкаталоги `src/`
Рекурсивно читай все `.rs` файлы:
```json
{
    "name": "powershell_cmd",
    "params": ["Get-ChildItem -Path \"C:\\путь\\к\\проекту\\src\" -Recurse -Filter *.rs | ForEach-Object { Write-Host (\"=== $($_.Name) ===\"); Get-Content $_.FullName -Encoding UTF8 }"]
}
```

### 3. ЕСЛИ ВОЗНИКЛИ ПРОБЛЕМЫ С КОДИРОВКОЙ
Используй PowerShell с явным указанием UTF-8:
```json
{
    "name": "powershell_cmd",
    "params": ["$old = [Console]::OutputEncoding; [Console]::OutputEncoding = [System.Text.Encoding]::UTF8; Get-Content \"C:\\путь\\к\\файлу.rs\" -Encoding UTF8; [Console]::OutputEncoding = $old"]
}
```

### 4. СОХРАНИ ПРОЧИТАННОЕ В СВОЙ КОНТЕКСТ
После чтения каждого файла:
- Кратко резюмируй его назначение
- Запомни ключевые структуры, функции, зависимости
- Отмечай связи между модулями

### 5. ФОРМИРУЙ СВОДКУ АРХИТЕКТУРЫ
После чтения всех файлов предоставь:
1. **Структура проекта** (основные модули и их роли)
2. **Зависимости** (из Cargo.toml и внешние крейты)
3. **Точки входа и потоки данных**
4. **Ключевые компоненты системы**
5. **Особенности протокола/API** (если есть)


## ⚠️ ВАЖНЫЕ ПРАВИЛА

1. **Читай модули по одному** - модули могут быть большими, на некоторых сайтах можно превысить допустимый размер вставки
   в поле ввода AI.
2. **Не читай весь состав каталогов просто на всякий случай** - там могут оказаться тысячи файлов, превысишь размер 
   вставки в поле ввода AI. Лучше прочти main.rs, начиная с него, выясни дерево модулей, потом забирай нужные файлы. 
3. **Фиксируй версии** — если есть `Cargo.lock`, прочитай и его
4. **Работай в два режима**:
    - **Режим захвата** — чтение файлов, сохранение в контекст
    - **Режим анализа** — построение ментальной модели системы
5**Работай самостоятельно** - обращайся к пользователю только, если для продолжения работы тебе нужна дополнительная 
    информация. Не обращайся просто за подтверждением продолжения работы, пользователь может остановить захват в случае 
    необходимости.
