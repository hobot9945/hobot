# Техническое задание: Расширение AI Native Bridge (Хобот)

## 1. Общие сведения
Расширение является связующим звеном между веб-интерфейсом ИИ (ChatGPT, DeepSeek, Google AI Studio) и нативным приложением Хобот. Основная задача — перехват директив из чата и их передача в постоянный поток `stdin` агента. Любые данные, генерируемые самим расширением (контекст, ошибки), передаются в специальных скобках.

## 2. Архитектура модулей
- **background.js**: Управление жизненным циклом нативного процесса и маршрутизация сообщений.
- **content.js**: Инициализация подсистем на странице и «заморозка» контекста.
- **dir_extractor.js**: Парсинг текстового потока, выделение директив и контроль очереди.
- **text_processor.js**: Очистка DOM-дерева от мусора и формирование чистого текстового потока.
- **focus_manager.js**: Принудительное удержание фокуса на поле ввода во время работы агента.

## 3. Управление нативным процессом (Native Messaging)
### 3.1. Постоянное соединение
- Использовать `chrome.runtime.connectNative` для открытия порта при полной загрузке страницы.
- При закрытии вкладки порт должен быть явно закрыт расширением.

### 3.2. Передача контекста (Формат расширения)
- Сразу после открытия порта расширение отправляет JSON-пакет, **заключенный в скобки расширения**: `<<<ext {JSON} >>>ext`.
- Структура пакета инициализации:
  - `type`: "INIT_CONTEXT"
  - `url`: текущий адрес страницы.
  - `window_title`: исходный заголовок окна браузера (для первичного поиска).
  - `unique_tab_title`: замороженный заголовок вкладки (title + уникальный суффикс `[HBT-xxxx]`). [file:10]

### 3.3. Отказоустойчивость
- При обрыве связи (`onDisconnect`) расширение предпринимает **ровно одну попытку** повторного подключения.
- В случае неудачи вывести пользователю системный `alert` с описанием ситуации.

## 4. Заморозка контекста вкладки
### 4.1. Стабилизация Title
- В `content.js` при старте генерируется 4-значный случайный ID (через `crypto.getRandomValues`), который добавляется в квадратных скобках к `document.title`.
- `MutationObserver` на узле `<title>` блокирует любые попытки сайта изменить заголовок, немедленно возвращая его к замороженному значению.

### 4.2. Обнаружение поля ввода
- Использовать справочник селекторов для поиска основного `textarea`.
- `FocusManager` должен автоматически переподключаться к новому элементу при SPA-навигации.

## 5. Управление состоянием занятости
- **Сброс флага**: Флаг `isAgentBusy` переводится в состояние `false` только после получения от Хобота специального сигнала завершения через `stdout` (нативный порт).
- **Разблокировка**: По факту гашения флага `FocusManager` прекращает принудительное удержание фокуса, а `AgentIface` получает разрешение на отправку следующей накопившейся директивы.

## 6. Обработка директив
### 6.1. Передача AI-директив
- Директивы, пришедшие от ИИ (`<<<ai ... >>>ai`), передаются в `stdin` Хобота в исходном виде без дополнительной обертки.

### 6.2. Директивы расширения
- Если расширение обнаруживает нарушение протокола (пропуск номеров, ошибки формата), оно формирует JSON с описанием ошибки.
- Этот JSON должен быть обязательно **заключен в скобки `<<<ext ... >>>ext`** перед отправкой Хоботу.
- Реализовать Mutex (`isAgentBusy`): блокировка отправки новых данных до получения отчета от Хобота.

## 7. Управление фокусом
- При активности агента (`isAgentBusy = true`) активируется режим жесткого удержания фокуса на элементе, найденном по селектору из `glob.js`.
- Перехват события `blur` выполняется на **capture-фазе** (для обхода стандартных обработчиков сайта) с принудительным возвратом фокуса через `setTimeout 0`.
- Блокировка переключения вкладок через `chrome.tabs.onActivated` в `background.js`.
