# start_work_on_hobot_agent.md

## Назначение
Этот промпт задаёт **минимальный захват Rust-проекта агента Hobot** из файловой системы Windows, чтобы:
- быстро восстановить структуру проекта;
- получить основу для дальнейшего точечного чтения модулей;
- не раздувать историю лишними файлами.

Предполагается, что **правила работы с Хоботом (формат директив, 2 режима)** уже загружены отдельным промптом.

## Среда
- Windows 10
- Rust 1.91
- IDE: RustRover
- Браузер: Chrome (Manifest V3 или V2 — уточняется по `manifest.json`)
- Промпты: `C:\Users\su144\RustroverProjects\rustdev\hobot\prompt`
- Установочные скрипты: `C:\Users\su144\RustroverProjects\rustdev\hobot\install`
- Проект: `C:\Users\su144\RustroverProjects\rustdev\hobot\web_extension\`
- Документация: `C:\Users\su144\RustroverProjects\rustdev\hobot\doc\`

## Общие правила захвата (строго)
1) Если не хватает контекста — **спрашивай**, не додумывай.
2) **Не читай лишнее** “на всякий случай”. Чтение — только по шагам ниже.
3) **Не лезь в артефакты сборки и мусор**:
    - `target/`, `.idea/`, `.git/`, логи (`*.log`), временные файлы.
4) **Тесты не читать**, если они не нужны для поставленной цели.
5) Цель захвата — структура и ключевая логика, а не полная индексация всего репозитория.

## Шаги минимального захвата (выполняй по порядку)
### Шаг 0 — Документация
Прочитай:
- `prompt/communication_style.md`
- `prompt/commenting_style.md`
- `doc/tech_spec/general_tech_spec.md`.

  Цель: понять назначение агента, ограничения, протокол/обмен и ключевые требования.

### Шаг 1 — Cargo.toml
Прочитай: `Cargo.toml`.

Цель: зафиксировать:
- имя crate, edition;
- ключевые зависимости и платформенную привязку (Win32/serde/toml/regex и т.д.).

### Шаг 2 — Структура каталогов (только папки)
Получить дерево **каталогов и файлов** для:
- `src/`
- `doc/`

Цель: увидеть где искать нужные модули.

### Шаг 3 — Точка входа и корень модулей
Прочитай: `install/hobot.bat.example`. Копия этого файла будет находиться в каталоге исполнения.

Цель:
- увидеть, что является точкой входа и как запускается агент;

Прочитай: `src/main.rs`.

Цель:
- увидеть, что является точкой входа и как запускается агент;
- извлечь **корень дерева модулей** (`mod ...` / `pub mod ...`).

### Шаг 4 — Выяснить полное дерево модулей (минимально)
Прочитай только те файлы, которые необходимы, чтобы собрать **полное дерево модулей**:
- идти от `main.rs` по `mod`-объявлениям;
- для каждого модуля подтвердить наличие соответствующего `.rs` файла или директории;
- не уходить глубоко в реализацию, если она не нужна для построения дерева.

Запреты на этом шаге:
- не читать тесты (`*_test.rs`, `tests/`) без явного запроса;

### Шаг 5 — Центральная часть: glob
Прочитай:
- `src/glob.rs`
- **все подмодули** `src/glob/*` (только то, что реально подключено через `mod ...`/`pub mod ...`).

Цель:
- понять глобальные константы/типы протокола;
- конфиг (где лежит, как грузится);
- централизованную обработку ошибок;
- транспорт stdout/stderr (Native Messaging), если он там.

### Шаг 6 — Общее представление о системе: agent
Прочитай: `src/agent.rs`.

Цель:
- понять главный цикл обработки;
- границы ответственности агента;
- основные переходы состояний (чтение → разбор → выполнение → отчёт).

## Результат после выполнения шагов 0–6 (что ты обязан выдать)
1) Дерево (с символами псевдографики) прочитанных модулей проекта.
2) Подгруженные доки и промпты.

## Дополнительно
Для захвата структуры каталогов используй команду `tree path /F /A`, для захвата файлов используй команду `type`.
