# Hobot: справочник команд

## Общее (важно)
- Всегда заполняй комментарии:
  - `dir_comment` — коротко: что делает директива.
  - `cmd_comment` — коротко: что делает конкретная команда и какие параметры ожидаются.
- Поле `name` — строка с именем команды.
- Поле `params` (если используется) — **массив строк**.
- Если команда **без параметров** — поле `params` **не указывать**.
- `allowed_timeout` (если используется) — таймаут выполнения команды в миллисекундах.

---

# Раздел: Shell / выполнение команд ОС

## Команда: `powershell_cmd`
**Назначение:** выполнить одну команду/скрипт через PowerShell.

**Параметры:**
- `params`: массив из **1 строки**
  - `params[0]`: командная строка PowerShell

**Примеры `params`:**
- `["Get-ChildItem \"C:\\Windows\" | Select-Object -First 20"]`
- `["(Get-Date).ToString(\"o\")"]`

---

## Команда: `type`
**Назначение:** вывести содержимое текстового файла.

**Важно:** это **команда `cmd.exe`**, поэтому используется **через** `shell_cmd` (в параметре `params[0]`).

**Параметры:**
- `params`: массив из **1 строки**
  - `params[0]`: командная строка `cmd`, содержащая `type "<path>"`

**Примеры `params`:**
- `["type \"C:\\Windows\\System32\\drivers\\etc\\hosts\""]`
- `["type \"C:\\Users\\su144\\RustroverProjects\\rustdev\\hobot\\Cargo.toml\""]`

**Примечания:**
- Путь к файлу почти всегда нужно брать в кавычки (`"..."`), чтобы не ломаться на пробелах.
- Команда подходит для чтения небольших файлов. Для больших файлов вывод будет очень длинным.

# Раздел: Управление окнами (win_control)

## Команда: `get_window_list`
**Назначение:** получить список top-level окон Windows, которые видит агент (через `EnumWindows`), с опциональными 
флагами включения “мусора”.

**Параметры:**
- параметры **опциональны**
- если параметры не заданы, вывод минимален - видимые окна с непустыми заголовками
- если параметры заданы:
  - `params`: массив строк (0..N)
  - порядок параметров **не важен**
  - поддержанные флаги:
    - `"include_invisible"` — включить невидимые окна (`IsWindowVisible=false`)
    - `"include_empty_title"` — включить окна с пустым заголовком

**Вывод:**
- Markdown-отчёт:
  - число окон
  - таблица: `#`, `x`, `y`, `width`, `height` ,`hwnd`, `title`
  - в конце `title` могут быть метки в скобках:
    - `(foreground)` — окно сейчас foreground
    - `(minimized)` — окно свернуто
    - `(invisible)` — окно невидимо (появляется только если включен `include_invisible`)

---
## Команда: `get_foreground_window_info`
**Назначение:** получить `HWND` и полный заголовок окна, которое в данный момент находится в фокусе (активно).

**Параметры:**
- параметров **нет** (`params` не указывать)

**Вывод:**
- Markdown-отчет, содержащий `HWND` (в hex-формате) и `title` активного окна.

---

## Команда: `find_window_info`
**Назначение:** найти окно по подстроке заголовка и вернуть его `HWND` и полный заголовок.  
Команда **НЕ** фокусирует окно и **НЕ** делает скриншот.

**Параметры:**
- `params`: массив из **1 строки**
  - `params[0]`: `"<needle>"` — подстрока для поиска в заголовке окна.

**Вывод:**
- Markdown-отчет с `HWND` и полным заголовком найденного окна. Если окон несколько, будет выбрано первое найденное.

---

## Команда: `focus_window_by_hwnd`
**Назначение:** сфокусировать (перевести в foreground) окно по его дескриптору `HWND`.

**Параметры:**
- `params`: массив из **1 строки**
  - `params[0]`: `"<hwnd>"` — дескриптор окна:
    - десятичное число (`"123456"`)
    - или hex в формате `"0x..."` (`"0x1A2B3C"`)

**Примечания:**
- Фокусировка выполняется с ретраями внутри `library::window::focus_window()`.
- При успехе команда возвращает отчёт с `hwnd` и `title` (заголовок окна).

---

## Команда: `focus_window_by_title`
**Назначение:** найти окно по подстроке заголовка и сфокусировать его.

**Параметры:**
- `params`: массив из **1 строки**
  - `params[0]`: `"<needle>"` — подстрока для поиска в заголовке окна.

**Примечания:**
- Команда использует `library::window::find_window_by_title_and_focus()` (поиск + фокусировка с ретраями).
- При успехе команда возвращает отчёт с `needle`, `hwnd` и `title`.
- Если окон несколько, будет выбрано первое найденное (как в `find_window_info`).

# Раздел: Захват экрана / мониторы (screenshot)

### Скриншоты и курсор мыши
Команды, возвращающие скриншоты (например: `capture_virtual_screen`, `capture_monitor`, `capture_region`, `capture_window_by_title`, `capture_window_by_hwnd`)
могут:
- **наложить курсор мыши** на изображение (если он был видим в момент захвата);
- дополнительно вывести в отчёт строку вида:
  - `Курсор: (x, y)` — если hotspot курсора попадает в область изображения;
  - `Курсор: (x, y) [вне изображения]` — если hotspot вне области изображения;
  - `Курсор: скрыт` — если курсор был невидим в момент захвата.

Координаты `(x, y)` — это **hotspot курсора** относительно левого верхнего угла захваченной области/изображения.

# Раздел: Эмуляция клавиатуры и вставка текста

## Команда: `paste_text_into_window_by_title`
**Назначение:** вставить текст в окно, найденное по подстроке заголовка (needle), через `clipboard + Ctrl+V` 
**с верификацией** (Ctrl+A/Ctrl+C).

**Параметры:**
- `params`: массив из **2 строк**
  - `params[0]`: `"<needle>"` — подстрока заголовка окна (поиск `contains`, регистр важен)
  - `params[1]`: `"<text>"` — текст для вставки

**Вывод:**
- Markdown-блок вида:
  - `needle='...'`
  - `hwnd=0x...`
  - `title='...'`
  - `len=<N>`

**Примечания:**
- Команда **сама** фокусирует найденное окно (best effort).
- Команда считает вставку успешной только если смогла **подтвердить** наличие текста в текущем поле ввода через 
  копирование (Ctrl+A → Ctrl+C).
- Поиск окна по needle ожидает **однозначное совпадение**: если найдено 0 или >1 окон — будет ошибка.
- Текст вставляется в **текущее поле ввода** внутри окна. Если в окне нет активного text-input (курсор не стоит в поле), 
  верификация, скорее всего, упадёт.

**Побочные эффекты:**
- Временно перезаписывает системный буфер обмена.
- Пытается восстановить буфер обмена **только для текстового содержимого** (если до команды в буфере было 
  изображение/файлы/HTML — восстановление “как было” не гарантируется).
- Генерирует события клавиатуры: `Ctrl+V`, `Ctrl+A`, `Ctrl+C`, стрелка вправо.

---

## Команда: `paste_text_into_window_by_hwnd`
**Назначение:** вставить текст в окно по `HWND` через `clipboard + Ctrl+V` **с верификацией** (Ctrl+A/Ctrl+C).

**Параметры:**
- `params`: массив из **2 строк**
  - `params[0]`: `"<hwnd>"` — дескриптор окна:
    - десятичное число (`"123456"`)
    - или hex в формате `"0x..."` (`"0x1A2B3C"`)
  - `params[1]`: `"<text>"` — текст для вставки

**Вывод:**
- Markdown-блок вида:
  - `hwnd_in='...'` (как пришло)
  - `hwnd=0x...` (нормализованный hex)
  - `len=<N>`

**Примечания / Побочные эффекты:**
- Те же, что у `paste_text_into_window_by_title`, только окно адресуется напрямую по `HWND` (менее двусмысленно для AI).

## Команда: `press_vk`
**Назначение:** нажать одну клавишу или комбинацию клавиш по виртуальным кодам (VK).

Команда предназначена для точечного управления клавиатурой, когда нужно явно указать VK-коды.
Если параметров больше одного, предполагается формат: **модификаторы + основная клавиша**.

**Параметры:**
- `params`: массив из **1..N строк**
  - если `params.len() == 1`:
    - `params[0]`: `"<vk>"` — виртуальный код клавиши (press)
  - если `params.len() >= 2`:
    - `params[0..N-2]`: `"<vk_modifier>"` — виртуальные коды модификаторов (Ctrl/Shift/Alt/Win и т.п.)
    - `params[N-1]`: `"<vk_key>"` — виртуальный код основной клавиши

**Формат VK-кода:**
- десятичное число: `"13"`, `"17"`, `"86"`
- или hex: `"0x0D"`, `"0x11"`, `"0x56"`

**Примеры `params`:**
- `["0x0D"]` — Enter
- `["0x11", "0x56"]` — Ctrl+V
- `["0x11", "0x10", "0x53"]` — Ctrl+Shift+S

**Вывод:**
- Markdown-блок с выполненным действием (что именно было нажато).

**Примечания:**
- Команда не гарантирует, что действие будет обработано приложением: это зависит от foreground окна и фокуса ввода.

---

## Команда: `press_key`
**Назначение:** нажать одну из “избранных” клавиш/комбинаций по строковому имени.

Это сокращённый и более безопасный вариант, чем `press_vk`, для часто используемых действий.

**Параметры:**
- `params`: массив из **1 строки**
  - `params[0]`: `"<key>"` — строка вида:
    - `"Ctrl+V"`
    - `"Ctrl+C"`
    - `"Ctrl+A"`
    - `"Enter"`
    - `"Right"` | `"Right_Arrow"`
    - `"Alt+F4"`
    - `bksp` | `"backspace"`
    - `del` | `delete`
    - `"Esc"`

**Нормализация входа:**
Перед распознаванием строка нормализуется:
- `trim()`
- `to_ascii_lowercase()`
- удаление пробелов `' '`

Например: `" Ctrl + V "` будет распознано как `"ctrl+v"`.

**Вывод:**
- Markdown-блок с выполненным действием (исходная строка из параметра).

**Ошибки:**
- Если комбинация не поддерживается — возвращается `Err(...)`.

**Примечания:**
- Команда не гарантирует, что действие будет обработано приложением: это зависит от foreground окна и фокуса ввода.

# Раздел: Эмуляция мыши (mouse)

## Команда: `get_mouse_position`
**Назначение:** узнать текущее положение курсора мыши в координатах виртуального рабочего стола.

**Параметры:**
- параметров **нет** (`params` не указывать)

## Команда: `mouse_move_to`
**Назначение:** плавно переместить курсор мыши в указанную точку (виртуальный рабочий стол).

**Параметры:**
- `params`: массив из **2 строк**
  - `params[0]`: `"<x>"` — координата X (может быть отрицательной)
  - `params[1]`: `"<y>"` — координата Y (может быть отрицательной)

**Примечания:**
- Перемещение выполняется **плавно** (траектория + easing), чтобы пользователю было комфортно наблюдать работу.
- Координаты — в пикселях виртуального рабочего стола (суммарное пространство всех мониторов).

---

## Команда: `mouse_move_by`
**Назначение:** плавно переместить курсор мыши на смещение `(dx, dy)` относительно текущей позиции.

Координаты и смещения — в пикселях виртуального рабочего стола (могут быть отрицательными).

**Параметры:**
- `params`: массив из **2 строк**
  - `params[0]`: `"<dx>"` — смещение по X (i32)
  - `params[1]`: `"<dy>"` — смещение по Y (i32)

**Примеры `params`:**
- `["10", "0"]` — сместить курсор на 10px вправо
- `["-50", "25"]` — сместить курсор на 50px влево и на 25px вниз

**Вывод:**
- `"OK"` (внутри markdown fence)

**Примечания:**
- Команда читает текущую позицию курсора (`get_mouse_position`), прибавляет смещение и выполняет плавное перемещение 
  в вычисленную точку.
- При вычислении целевой позиции есть защита от переполнения i32 (в нормальной работе не должно происходить).

## Команда: `mouse_scroll`
**Назначение:** вертикальная прокрутка колёсиком мыши (scroll wheel). Опционально — сначала плавно подвести курсор к нужной точке.

**Параметры:**
- `params`: массив из **1 строки**
  - `params[0]`: `"<lines>"` — количество “шагов” прокрутки:
    - `> 0` — прокрутка вверх
    - `< 0` — прокрутка вниз
- **или**
- `params`: массив из **3 строк**
  - `params[0]`: `"<x>"` — координата X (виртуальный рабочий стол)
  - `params[1]`: `"<y>"` — координата Y (виртуальный рабочий стол)
  - `params[2]`: `"<lines>"` — количество “шагов” прокрутки

**Примечания:**
- Если заданы `x,y` — сначала выполняется **плавное перемещение** курсора в точку, затем прокрутка.
- Один “шаг” соответствует `WHEEL_DELTA = 120` в WinAPI. Реальная величина прокрутки зависит от настроек Windows и приложения.

---

## Команда: `mouse_drag`
**Назначение:** перетаскивание (drag-and-drop) объекта мышью из точки A в точку B.

**Параметры:**
- `params`: массив из **4 строк**
  - `params[0]`: `"<x_from>"` — X точки захвата
  - `params[1]`: `"<y_from>"` — Y точки захвата
  - `params[2]`: `"<x_to>"` — X точки сброса
  - `params[3]`: `"<y_to>"` — Y точки сброса

**Примечания:**
- Реализация выполняет **плавное** перемещение в точку захвата, нажимает ЛКМ, делает небольшой “срыв” курсора (чтобы гарантировать старт drag), затем плавно ведёт к точке сброса и отпускает ЛКМ.
- Координаты — в пикселях виртуального рабочего стола (могут быть отрицательными).

# Раздел: Утилиты / панель задач (misc_tool)

## Команда: `show_taskbar`
**Назначение:** сделать панель задач видимой (отключить режим Auto-hide).  
Состояние панели задач сохраняется во внутренней переменной. Впоследствии его нужно восстановить командой `restore_taskbar_autohide_state`.

**Параметры:**
- параметров **нет** (`params` не указывать)

**Примечания:**
- Команда запоминает состояние панели задач перед изменением. Используйте `restore_taskbar_autohide` для возврата.

---

## Команда: `restore_taskbar_autohide_state`
**Назначение:** восстановить состояние панели задач (вернуть Auto-hide), которое было до вызова `show_taskbar`.

**Параметры:**
- параметров **нет** (`params` не указывать)

---

## Команда: `show_taskbar_autohide_state`
**Назначение:** проверить, включен ли режим авто-скрытия (Auto-hide) панели задач.

**Параметры:**
- параметров **нет** (`params` не указывать)

**Вывод:**
- Строка `"true"` (если Auto-hide включен) или `"false"` (если выключен).
