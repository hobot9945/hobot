## 3. Существенные данные первички, которые не попали в `primary.md`

- **Соотношение «рулоны → пакеты» в Инвойсе и PL.** В Инвойсе товары указаны в рулонах (60, 30, 60, 30, 90, 180, 5), а в PL — в пакетах (60, 30, 6, 3, 9, 18, 1). Коэффициенты: 1:1 для Антикот, 10:1 для Антипыльца и Антимошка, 5:1 для Трехслойной сетки. Данные существенны для корректного заполнения граф ДТ (количество в ДТ должно соответствовать выбранной единице измерения). Не формализовано, т.к. в primary.md нет отдельного поля для коэффициента пересчёта.

- **CMR содержит только один код ТН ВЭД (5804101000)** в единственной товарной строке, хотя фактически груз включает два кода: `5804101000` и `7019900095`. Не формализовано, т.к. CMR не предусматривает детализации по кодам — указан основной код.

- **CurrencyRate = 10.9430** (курс CNY/RUB) указан в Invoice как `confirmed_operator`, но значение отсутствует во всех документах поставки. Данные существенны для расчёта таможенной стоимости. Не формализовано, т.к. нет документального подтверждения значения.

- **Вариации наименования продавца** в разных документах: `HEBEI LANGMAI IMPORT AND EXPORT CO., LTD` (контракт), `HEBEI LANGMAI IMPORT AND EXPORT CO.,LTD.` (инвойс, PL), `HEBEEI LANGMAI IMPORT & EXPORT CO., LTD.` (CMR). Не формализовано как отдельное поле, т.к. для Альты ключевое значение имеет ИНН/наименование контракта, а не написание.

- **Вариации наименования экспедитора** в счетах: `ООО «Трансмипериал»` (счет за перевозку), `ООО «Транснипериал»` (счет за страховку), `ООО «Трансимпериал»` (договор). Верное написание по договору: `ООО «Трансимпериал»`. Не формализовано, т.к. в primary.md значения взяты из документов как есть.
