# dt_xml_review_schema.md — Review этапа 2.1 (DT XML)

## Назначение

Файл `dt_xml_review.md` фиксирует результат генерации XML ДТ на этапе 2.1 и выявленные технические проблемы.

Этап 2.1 считается завершенным только если:
- `dt.xml` успешно сформирован и записан в `stage_2.1_result\\<кейс>\\`;
- XML корректно читается и well-formed;
- структура соответствует `dt_xml_schema.md` (whitelist-теги, повторяющиеся блоки);
- нет нерешённых блокеров, мешающих импорту в Альту.

## 4. Проверка структуры и соответствия схеме

Проверить для `dt.xml`:

### 4.1 Формальные проверки
- Корневой тег: `<AltaGTD>`
- XML well-formed (нет синтаксических ошибок)
- Кодировка: `windows-1251`
- Все значения XML-экранированы

### 4.2 Структурные проверки (whitelist)
- Присутствуют все обязательные теги верхнего уровня из `dt_xml_schema.md`:
  - `G_1_*`, `G_2_*`, `G_5_*`, `G_6_*`, `G_8_*`, `G_9_*`, `G_11_1`, `G_14_*`, `G_15*..G_17*`, `G_18*..G_21*`, `G_20_*`, `G_22_*`, `G_23_*`, `G_25_1`, `G_26_1`, `G_29_*`, `G_30_*`, `G_42_2`, `G_54_*`, `G_54P/N2`
- Количество `<BLOCK>` равно числу `goods[]` в `dt_fields.yaml`
- Для каждого `<BLOCK>`:
  - есть `<G_31>` и поля `G_32_1..G_46_1` согласно схеме
  - количество `<TOVG>` равно `len(goods[i].tovg)`
  - количество `<TXT>` равно `len(goods[i].txt)`
  - если включены платежи: теги `G_47_<n>_<m>` сформированы последовательно

### 4.3 Соответствие маппинга dt_fields → XML
Выборочно проверить 5–10 ключевых полей:
- `G_1_1` ← `declaration.direction`
- `G_2_NAM` ← `sender.name`
- `G_6_1` ← `shipment.total_packages`
- `G_22_2` ← `shipment.invoice_amount`
- `BLOCK/G_33_1` ← `goods[i].tnved_code`
- `BLOCK/TOVG/G31_1` ← `goods[i].tovg[j].description`
- `G_54_20` ← `representative.date`
