# Инструкция по этапу 0: Преобразование первичных документов в формат маркдауна

AI подготавливает первичные документы к обработке этапа 1: приводит doc, docx, xls, xlsx, pdf, образы к 
текстовому виду (md). На этом этапе AI не извлекает факты поставки и не интерпретирует содержимое документов.



### Требования и правила
Если первичный документ уже в формате md (редкий случай), он просто копируется в каталог `md`.

- Первичными документами считаем только те, которые нужны для составления ДТ. Эталоны, явный мусор отбрасывай.
  Если в сомнении - включай.

- **СОХРАНЯЙ ДОКУМЕНТЫ ОДИН-В-ОДИН. ЗАПРЕЩЕНО СОКРАЩАТЬ ДОКУМЕНТЫ. ПЕРЕСКАЗ ЗАПРЕШЕН! ПОТЕРЯ ДАННЫХ КРИТИЧНА!:**
  сохраняй содержание документов, держи форму результата близко к оригиналу.

- **ЕСЛИ ДОКУМЕНТ ПРОЧИТАН НЕ ПОЛНОСТЬЮ, ЗАПРЕШАЕТСЯ ЕГО КОНВЕРТАЦИЯ БЕЗ РАЗРЕЩЕНИЯ ОПЕРАТОРА**. Такое может случиться,
  например, при распознавании текста из PDF или файлов образов. В этом случае, AI обязан остановиться, уведомить
  оператора и запросить инструкции по продолжению работ.

- Файлы с одинаковыми именами и разными расширениями представляют один и тот же документ.

- **ПРИОРИТЕТ ФОРМАТОВ. ПЕРВЫМИ ПРОБУЙ ЛЕГКООБРАБАТЫВАЕМЫЕ ФОРМАТЫ.** Если документ представлен в нескольких форматах, 
  выбирай самый удобный. Приоритет:
  - Если документ в текстовом формате (txt, md, xml), это самый лучший, но редкий вариант.
  - Если сайт принимает файлы Docx, Xlsx, то предпочитай эти форматы.
  - Если есть образ или набор образов документа (часто документ не помещается в один скриншот). Это PNG и другие форматы.
  - PDF, если нет других вариантов. Его обработка тяжелее и менее надежна.

- Если документ многофайловый, например, представлен несколькими PNG образами, нужно сначала взять все образы, и
  только потом формировать итоговый документ. **Не следует** после взятия каждого образа формировать/доформировывать md.

- Если данные в нечитаемых форматах, AI должен попросить оператора преобразовать их в читаемый вид.

- Если pdf превышает лимит ввода (например, для arena.ai 15Mb), оператор должен поместить скриншоты страниц в файлы
  образов, индексировав имя документа номером страницы. Перед требованием скриншотов предложи оператору для проверки
  вручную переместить файл в поле ввод чата.

- Файлы, не требующие преобразования, просто копируются в каталог `md`.

- Файлы xml, это, обычно, выгрузки Альты, записаны в кодировке cp-1251.

- Если `md` заполнен частично, продолжить работу по его заполнению.

- Если какие-то первичные документы не были достоверно прочитаны, повторить.

- **ЗАПРЕШАЕТСЯ ПРЕЖДЕВРЕМЕННО ЗАКАНЧИВАТЬ КОНВЕРТАЦИЮ,** скажем, из соображений, что уже собрано достаточно данных.
  Задача нулевого этапа не оценка, а сбор данных. Любые файлы, которые могут рассматриваться как полезные, должны
  быть приняты.

### Папки:

- Первичные документы лежат в `alta\source\<кейс>\...`. Где-то внутри лежит каталог поставки. Рекурсивно прочти 
  `alta\source\<кейс>`, выясни путь к каталогу поставки.

- Результаты этапа 0 помещаются в папку `<каталог поставки>\md`:
  - `*.md` — текстовые версии документов (по одному файлу на исходный документ)
  - `doc_conversion_review.md`. Составляется по окончании работы. Содержит:
    - таблицу соответствия названий документов исходным файлам;
    - оценку полноты данных для целей формирования ДТ;
    - вопросы и замечания, если есть;

### ЧЕКЛИСТ:

**После записи каждого документа** нужно:
- вспомнить этот промпт
- сравнить сведения исходного документа и полученного md, **УБЕДИТЬСЯ ЧТО ВСЕ ДАННЫЕ ПЕРЕНЕСЕНЫ, НИЧЕГО НЕ ПОТЕРЯНО**.
- если обнаружена потеря, перегенерировать md-файл.

**После завершения преобразования** выполнить проверки:
- сверить `doc_conversion_review.md` с содержимым каталога `md`;
- сверить состав первичных документов с `doc_conversion_review.md` - все ли документы обработаны; **ПОТЕРЯ ИНФОРМАЦИИ КРИТИЧНА!**

### Разрешения:
- **РАЗРЕШЕНО ЧТЕНИЕ ИЗ `alta\source\<кейс>\...`**
- **РАЗРЕШЕНА ЗАПИСЬ В `<каталог поставки>\md\...`**
