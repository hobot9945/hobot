# Альта-ГТД: заметки по навигации в режиме списка ДТ

## Контекст

Исследование велось в приложении **«Альта-ГТД» v. 2.0.261.40**. 
Рабочий режим: **`Список SQL-ДТ (ГТД)`**. 
Интересующий режим документов: **`ДТ (ГТД)`** — соответствующая вкладка в верхней панели должна быть активна.

## Что означает ГТД

`ГТД` — это старое название: **грузовая таможенная декларация**. 
В текущей практике по смыслу это соответствует режиму работы с **ДТ**, но в интерфейсе и в речи пользователей старое обозначение все еще используется.

## Общая структура окна

Главное окно Альты в этом режиме состоит из трех ключевых зон:

### 1. Левая панель

Это дерево внутренних каталогов Альты.
Верхний корневой узел:
`Основной каталог`
Внутри видны системные и пользовательские папки, например:
`Корзина`, `Избранные`, `Шаблоны`, `ВНТ`, `БентоКор`, `БентоПро`, `ВИС`, `Гари`, `Иннотек`, `КРЗЦ`, `Леповер`, `Массажеры`, `не актуальные`, `расчеты`, `Симанитрон`, `Симанитрон-Электроника`, `СКИФ`, `Энергосбережение`.
Для ветки `СКИФ` были замечены вложенные папки:
`Запчасти для термометрс`, `Сетки`, `ТМП`.
Внизу левой панели есть чекбокс:
`Все папки вместе`.

### 2. Центральная таблица

Это основной рабочий список объектов текущего каталога.
Именно по этой таблице удобнее выполнять навигацию, чем по левому дереву.
Таблица реагирует на клавиши:

- стрелки вверх/вниз;

- `Enter` — вход в папку;

- `Backspace` — возврат на уровень выше.
  Практический вывод: для дальнейшей автоматизации основной упор надо делать именно на центральную таблицу.
  
  ### 3. Нижняя панель
  
  Это панель детализации. В ней есть вертикальные вкладки:
  `Инфо`, `Заголовок`, `Взаимосвязанные`, `Товары (Ctrl+G)`.
  Для текущей задачи она пока вторична.
  
  ## Главное наблюдение по фокусу
  
  Для того чтобы клавиатурная навигация работала, **фокус должен быть установлен именно в центральную таблицу**.
  Недостаточно просто сфокусировать окно Альты целиком. 
  Нужно сначала сделать клик мышью по строке в центральной таблице, и только после этого отправлять клавиши.
  Рабочая схема:
1. найти окно Альты;

2. перевести мышь в центральную таблицу;

3. сделать левый клик по строке;

4. только потом отправлять стрелки и `Enter`.
   Это ключевой момент, который надо учитывать в следующей сессии с самого начала.
   
   ## Что удалось подтвердить по таблице
   
   После клика в таблицу и одной стрелки вниз выделение реально сдвинулось:
- было `Корзина`;

- стало `Шаблоны`.
  Значит:

- центральная таблица действительно принимает клавиатурную навигацию;

- базовая логика переходов подтверждена;

- проблема не в самой Альте, а в необходимости правильно захватывать рабочий фокус перед серией действий.
  
  ## Навигация по списку папок
  
  В корневом каталоге порядок строк в центральной таблице наблюдался такой:
1. `Корзина`
2. `Шаблоны`
3. `ВНТ`
4. `БентоКор`
5. `БентоПро`
6. `ВИС`
7. `Гари`
8. `Иннотек`
9. `КРЗЦ`
10. `Леповер`
11. `Массажеры`
12. `не актуальные`
13. `расчеты`
14. `Симанитрон`
15. `Симанитрон-Электроника`
16. `СКИФ`
17. `Энергосбережение`
    Из этого следует:
- от `Корзина` до `СКИФ` нужно **15 нажатий вниз**;
- ## Что известно о пути к целевой папке
- Целевая папка:
  `СКИФ/ТМП`
  Структура ветки `СКИФ`, замеченная в дереве:
- `Запчасти для термометрс`
- `Сетки`
- `ТМП`
  То есть после входа в `СКИФ` нужно будет отдельно перейти на `ТМП` и войти в нее.
  Ожидаемая логика:
1. из корня дойти до `СКИФ`;

2. `Enter` — войти в `СКИФ`;

3. внутри `СКИФ` выбрать `ТМП`;

4. `Enter` — войти в `ТМП`.
   
   ## Что удалось понять про левую панель
   
   Хотя основной путь лучше строить через таблицу, левая панель тоже частично исследована.
   Она реагирует в первую очередь на мышь и имеет собственное контекстное меню. 
   В этом меню были замечены пункты:
- `Удалить каталог`

- `Создать каталог`

- `Переименовать каталог`

- `Установить корневой каталог`

- `Добавить в панель быстрого доступа`

- `Показать панель быстрого доступа`

- `Показывать только документы пользователя`

- `Обновить дерево`

- `Спрятать дерево...`

- `Новый режим папок`

- `Полностью раскрывать дерево`
  Практический вывод:
  левая панель — это не просто визуальный индикатор, а отдельный управляемый элемент интерфейса с собственными режимами отображения и операциями над каталогами.
  Но для дальнейшей автоматизации лучше использовать ее только как вспомогательный ориентир, а не как основной способ переходов.
  
  ## Что известно про меню «Каталог»
  
  В главном меню есть пункт `Каталог`, а в нем подпункты вроде:

- `Список ДТ`

- `Список ДТС-1`

- и другие подобные.
  Рабочая гипотеза:
  это не “открытие файловой системы”, а переключение типа списка или фильтра по виду документов.
  Это согласуется с верхними вкладками:

- `ДТ (ГТД)`

- `ДТС-1/3`

- `ДТС-2/4`

- `КДТ`

- `Опись`

- `КУТС`

- `ТД/ПИ`

- `ЭД`

- `ПРОЧИЕ`
  То есть вероятная схема такая:

- левая панель задает каталог;

- центральная таблица показывает содержимое;

- верхние вкладки и пункт `Каталог` задают тип или фильтр документов.
  Это пока гипотеза, но она выглядит правдоподобной и полезной для следующей сессии.
  
  ## Практические правила для следующего захода
1. Работать через центральную таблицу, не через левое дерево.

2. Каждую директиву с клавишами начинать с установки фокуса:
   
   - найти окно Альты;
   - нажать Shift+F3, это меню Каталог/Список ДТ. Высветится окно с панелью навигации слева и таблицей в центре.
   - перевести мышь на пункт "Корзина" боковой панели, кликнуть по нему, потом кликнуть по пункту "Основной каталог". Это приведет таблицу в исходное состояние с курсором на пункте "Корзина".
   - После длинной серии действий проверять состояние скриншотом.

3. Координаты мыши считать **относительно окна Альты**, не от экрана.

4. Для перехода к `СКИФ/ТМП` использовать таблицу как основной навигатор.
   
   ## Краткий итог
   
   За эту сессию удалось зафиксировать не случайные наблюдения, а несколько устойчивых правил:
- режим `ДТ (ГТД)` — нужный рабочий контекст;
- центральная таблица — главный объект для навигации;
- клавиатурные переходы в таблице работают при корректно установленном фокусе;
- путь к нужной ветке проходит через каталог `СКИФ`, внутри которого есть `ТМП`;
- левая панель имеет собственные режимы и контекстное меню, но как основной канал управления менее удобна.
