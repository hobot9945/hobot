# doc_xml_review_schema.md — Review этапа 1.1 (Формализация XML)

## Назначение
Файл `doc_xml_review.md` фиксирует результат генерации XML-документов на этапе 1.1 и выявленные проблемы
(как технические, так и входные — pending/неполнота в primary.md).

Этап 1.1 считается завершенным только если:
- все обязательные документы сформированы;
- XML-файлы успешно записаны в `stage_1.1_result\<кейс>\formalized_docs\`;
- не осталось нерешённых блокеров (pending по формализуемым полям документов, которые генерируются в XML).

## 2. Проверка входных данных

### 2.1 Статус primary.md

- Есть ли pending в документах `formalized`, которые должны быть преобразованы в XML?
    - Если да — перечислить критичные:
        - `<UQI поля>`
    - Решение:
        - ☐ Остановлено (возврат к этапу 1.0)
        - ☐ Продолжено в режиме отладки

Примечание:
- Stage 1.1 не должен “додумывать” значения pending-полей. В рабочем режиме pending по формализуемым полям = блокер.

## 4. Проверка структуры и переноса данных

Для каждого созданного файла проверить:
- Корневой тег соответствует `xml_target_root` (из `primary.md`)
- XML валиден (без синтаксических ошибок)
- Все поля из `primary.md/field` перенесены (с учетом правил скаляров/объектов/массивов)
- Даты приведены к формату `YYYY-MM-DD`
- Кодировка `windows-1251`

Если обнаружены расхождения — перечислить:

- `<имя файла>`
    - проблема: <описание>
    - источник: <UQI поля из primary.md>
    - причина: <если понятна>
    - статус: исправлено / не исправлено
