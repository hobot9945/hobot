# Инструкция для AI по работе с декларациями в Альте

## 0. Этапы работы

Этапы зависимы друг от друга только в направлении от раннего к позднему. Промпты следующего этапа не нужны для работ
текущего этапа.  

## Этап 0.

AI подготавливает первичные документы к обработке этапа 1: приводит pdf и образы к текстовому виду (md).
На этом этапе AI не извлекает факты поставки и не интерпретирует содержимое документов.

### Требования и правила 
Первичные документы могут поступать в форматах pdf, png, jpg, txt, xml, md.
- Первичными документами считаем только те, которые нужны для составления ДТ. Эталоны, явный мусор отбрасывай. 
  Если в сомнении - включай.
- **ЗАПРЕЩЕНО СОКРАЩАТЬ / МОДИФИЦИРОВАТЬ / АГРЕГИРОВАТь ДОКУМЕНТЫ:** содержание документов должно полностью сохранено,
  форма близка к оригиналу.
- Файлы с одинаковыми именами и разными расширениями представляют один и тот же документ.
- Если данные в нечитаемых форматах, AI должен попросить оператора преобразовать их в читаемый вид.
- Если сайт принимает файлы Docx, Xlsx, то предпочитай эти форматы, потом образы, потом pdf.
- Если pdf превышает лимит ввода (например, для arena.ai 15Mb), оператор должен поместить скриншоты страниц в файлы 
  образов, индексировав имя документа номером страницы. Перед требованием скриншотов предложи оператору для проверки 
  вручную переместить файл в поле ввод чата.
- Файлы, не требующие преобразования, просто копируются в каталог `md`.
- Файлы xml, это, обычно, выгрузки Альты, записаны в кодировке cp-1251.
- Если `md` заполнен частично, продолжить работу по его заполнению.

### Чеклист:
После завершения преобразования и построения index.md выполнить проверки:
- сверить `reviw_0.md` с содержимым каталога `md`;
- сверить состав первичных документов с index.md - все ли документы обработаны; **ПОТЕРЯ ИНФОРМАЦИИ КРИТИЧНА!**

Исходные данные лежат в:

- `alta\source\<каталог поставки>\` — первичные документы
- `alta\stable_source` — общие для кейсов документы, их не конвертируем

Результаты этапа 0 помещаются в папку `alta\source\<каталог поставки>\md`:

- `*.md` — текстовые версии документов (по одному файлу на исходный документ)
- `review_0.md`. Составляется по окончании работы. Содержит: 
  - таблицу соответствия названий документов исходным файлам;
  - оценку полноты данных для целей формирования ДТ;
  - вопросы и замечания, если есть; 

**РАЗРЕШЕНА ЗАПИСЬ В `alta\source\<каталог поставки>\md`**

## Этап 1.

AI разбирает первичные документы по поставке, извлекает и сверяет факты. При необходимости AI переходит в режим диалога
с оператором для сбора недостающих фактов и разрешения конфликтов.
На основе собранных данных и шаблонов формализации AI формирует набор формализованных документов для:
- импорта формализованных первичных документов в Альту
- файл формализованных исходных данных для этапа 2
- отчет по этапу 1 для оператора
- возможно, файл, содержащий сведения, полученные от оператора (может пригодиться для повторных прогонов)

Исходные данные лежат в:

- `alta\source\<каталог поставки>\md` первичные документы
- `alta\stable_source` документы, общие для кейсов. Например, данные декларанта.
- При недостатке данных в `md`, могут выполняться обращения к первичным документам. Такие факты фиксируются в ревью.

Шаблоны и правила лежат в `alta\prompt\stage_1`:

- `primary_schema.md` извлеченные из первички факты, а также пояснения оператора.
- `doc_xml_schema.md` мапирование полей документов из `primary_schema.md` в xml-теги импортируемых формализованных
  документов.
- `review_1_schema.md` шаблон отчета по этапу 1

Подготовленные исходные данные лежат в `alta\source\<каталог поставки>\md`, их использование в приоритете.

Результаты этапа 1 лежат в папке `alta\stage_1_result\<имя каталога поставки>\`:

- `primary.md` факты поставки, собранные из первички и готовые для этапа 2 и генерации импортируемых формализованных
  документов.
- `formalized_docs` каталог, в который помещаются xml файлы формализованных документов, предназначенных для импорта в Альту
- `review_1.md` - отчет по этапу 1
- `alta\source\<каталог поставки>\operator\operator_provided_data.md` - опционально, данные, полученные от оператора (из диалога с оператором, 
  если у AI возникли вопросы)

## Этап 2

Исходные данные лежат в папке `alta\stage_1_result\<имя каталога поставки>\`:

- `primary.md` 

Шаблоны и правила лежат в `alta\prompt\stage_2`:

- `dt_fields_schema.md` - шаблоны для подготовки `dt_fields.md` из первичных данных `primary.md`
- `review_2_schema.md` - схема отчета по этапу 2

Результаты этапа 2 лежат в папке `alta\stage_2_result`:

- `dt_fields.md` данные, подготовленные для генерации `dt_xml.md`
- `review_2.md` отчет по этапу 2

## Этап 3

Исходные данные лежат в `alta\prompt\stage_2_result\dt_fields.md`

Шаблоны и правила лежат в `alta\prompt\stage_3`:
- `dt_xml_schema.md` промежуточный промпт для генерации `dt.xml`.

Результаты этапа 3 лежат в папке `alta\stage_3_result\<имя каталога поставки>\`:
- `dt.xml`, пригодный для импорта в Альту`ui_input.md` — опционально, карта ввода ДТ в Альту. Следует формировать в кодировке `cp1251`.

## Этап 4

- Ручной или автоматический импорт `doc_xml_schema.md` в Альту.
- Ручной или автоматический импорт `dt.xml` в Альту.

После завершения любого этапа возможно и желательно очистить историю чата.

---

## 1. Базовые правила

### 1.1. Никакой отсебятины

Любое значение, попадающее в `primary.md`, должно быть:

- извлечено из первичных документов;
- либо подтверждено оператором;
- либо корректно выведено из подтвержденных данных;
- либо явно помечено как эталонное наблюдение, не являющееся фактом новой поставки.
  Если данных не хватает — остановиться и обратиться к оператору либо явно зафиксировать пробел.

## Общение с оператором

### Один вопрос за раз

Если требуется вмешательство оператора, AI задает вопросы по одному, с описанием проблемы и полными ссылками на 
источники (имена файлов, поля и т.д.)

# ПОРЯДОК РАБОТЫ

- прочитай структуру каталога `alta`;
- выйди в режим диалога, запроси у оператора название кейса и номер этапа; запроси, позволяет ли сайт перетаскивать в
  поле ввода чата файлы Docx, Xlsx;
- прочитай промпты, для текущего этапа;
- начинай работу по этапу;

### Примечание
- Для загрузки docx, xlsx, pdf, png используй перетаскивание файлов. При превышении объема 15 Мб реши вопрос возможности
  перетаскивания с оператором. pdf, png можно перетаскивать пачками. 
- Для доступа к текстовым файлам используй команды Хобота `read_file`, `write_file`
- При загрузке пачками текстовых файлов смотри чтобы суммарный размер не превышал 120 кбайт.
